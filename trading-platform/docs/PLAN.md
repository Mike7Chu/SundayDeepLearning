# 트레이딩/아비트라지 플랫폼 — 승인된 계획서

> 이 문서는 사용자가 승인한 전체 계획(원본은 세션 plan 파일). 새 세션은 이 문서를 단일 기준으로 삼는다.

## Context (왜 만드는가)

두 축을 가진 개인 트레이딩 플랫폼을 라즈베리파이4(OMV, 8TB, 24h)에 구축한다.

**(A) 코인 아비트라지 시스템** — 9개 거래소(업비트·빗썸·바이낸스·바이비트·MEXC·비트겟·게이트·BingX·OKX)의 가격차(김프/역프)·펀딩비(펀비)를 실시간 모니터링. **알림이 최우선 핵심**이고, 그 위에 더따리(theddari) 스타일 대시보드 + 봇들로 운용:
- **알림봇 (최우선)**: 김프/역프/펀비 임계치 실시간 알림 — 시스템의 핵심.
- **공지알림봇**: 업비트/빗썸 공지(신규상장 등) 실시간 전달 → 신규상장 갭 조기 포착(현선봇 트리거).
- **현선봇**: 공지로 신규상장 포착 시 현물-선물을 즉시 동시 진입해 헤지/차익(델타뉴트럴, 펀비·갭 캡처).
- **loan봇**: 진입 자금 확보를 위해 차입(loan/마진).
- **매도봇**: 갭이 좁혀지면 내 보유잔고로 매도(청산/익절).
- 위 봇들을 **텔레그램 + 대시보드 양쪽에서 on/off·파라미터 제어**.

**(B) 주식 시스템** — 세 전략을 각각 대시보드로 관리 + (게이트 통과 후) 매매. 정기적으로 정해진 규칙에 따라 자동 매매하되, 추천 퀀트 알고리즘도 제안:
- **시그널 기반 매매** (기술적/퀀트 시그널)
- **가치투자** (펀더멘털 스크리닝)
- **배당주 모으기** (배당 캘린더 + 정기 적립 매수)
- **텔레그램 브리핑 + 뉴스 요약(LLM)**: 보유/관심종목 일일 브리핑, 관련 뉴스 요약.

### 사용자 결정사항 / 원칙
- **둘 다 병행**, **모니터링·알림을 먼저 검증한 뒤 실행봇을 단계적으로 켠다**(실주문은 안전장치 + 소액부터).
- 기술 스택은 추천에 맡김 → **라즈베리파이 올인** 제약에 맞춰 경량 컨테이너 스택.

## 추천 기술 스택 (RPi4 친화적)
- **데이터 수집**: Python + `ccxt` — 9개 거래소 단일 인터페이스(현물·호가·펀딩비·입출금·마진/차입). WS는 ccxt(pro) 내장, 미지원은 REST 폴링.
- **백엔드/API**: FastAPI(async) + WebSocket fanout.
- **실시간 캐시/펍섭**: Redis — 최신 시세 + 수집기↔API↔봇 메시지 버스 + rate-limit 조율 + 봇 상태.
- **시계열 저장**: TimescaleDB(PostgreSQL, ARM64) 또는 경량화 시 SQLite(핫)+Parquet(콜드). **저장 예산 ~2TB 상한**(8TB 중; 보존기간·압축·다운샘플로 관리).
- **프론트엔드**: Next.js + TypeScript, 실시간 WS, `lightweight-charts`. 더따리식 정렬/필터 테이블 + 봇 컨트롤 패널.
- **알림/제어**: 텔레그램 봇(python-telegram-bot) — 알림 + 명령 제어.
- **주식 API**: 한국투자증권 OpenAPI(KIS) — Linux/Python REST+WS 지원. 모의투자 계좌로 시작.
- **인프라**: Docker Compose(OMV-Extras/Portainer), 외부접속 Cloudflare Tunnel/Tailscale, 리버스프록시 Caddy.

### RPi4 성능 가드레일
- 공통 상장 상위 N개 코인으로 심볼 유니버스 제한(설정값), 점진 확대.
- 컨테이너 메모리 제한, TimescaleDB 압축 정책.

## 핵심 설계: 봇 컨트롤 플레인 (공통 프레임워크)
모든 봇(현선/loan/매도/주식 전략)은 동일 패턴 공유:
- **Strategy**(조건 판단) → **Executor**(주문) → **State machine**(idle/armed/running/stopped) → **Control API**.
- 텔레그램과 대시보드는 둘 다 Control API 호출(단일 진실원). on/off, 파라미터, 상태/체결/PnL.
- **Execution Gateway**: 모든 실주문 게이트웨이 경유 → 글로벌 킬스위치, 포지션/명목 한도, 기본 dry-run(페이퍼), 출금권한 없는 API 키, IP 화이트리스트.

```
├── api/         (services/ routers/ main.py)

└── shared/      (공통 스키마)
```

## 단계별 로드맵
- **Phase 0 — 기반 셋업**: OMV Docker/Portainer, 스캐폴딩, docker-compose, .env, 시크릿 규칙, (선택)터널. ✅
- **Phase 1 — 코인 수집 MVP**: ccxt 9거래소 ticker + 환율 → Redis/DB. 심볼 유니버스. ✅
- **Phase 2 — 알림(최우선) + 대시보드 코어**: 김프/역프 순스프레드, 알림봇 + 공지알림봇(읽기전용), 프리미엄/펀비 대시보드. ⏳(계산/API ✅, 알림봇·대시보드·펀비 미구현)
- **Phase 3 — 봇 컨트롤 플레인 + 페이퍼**: framework + execution_gateway(dry-run), 현선/loan/매도 페이퍼, 텔레그램+대시보드 제어.
- **Phase 4 — 코인 실행봇 활성화**: loan 차입 메커니즘 확정, 실거래 게이트 단계 개방, 한도·킬스위치, 매도봇 실보유잔고 청산.
- **Phase 5 — 주식 모니터링/대시보드 + 브리핑**: KIS(모의) 시세/펀더멘털/배당, 대시보드 3종, 텔레그램 일일 브리핑 + 뉴스 요약(LLM).
- **Phase 6 — 주식 실행(gated, 정기 규칙)**: 스케줄 규칙 매매(시그널/가치/배당 DRIP), 모의→실계좌.

## 추천 퀀트 알고리즘 (주식, 출발점)
- **시그널**: 듀얼 모멘텀(절대+상대), 이동평균 골든/데드크로스, RSI·볼린저 평균회귀.
- **가치투자**: 마법공식(이익수익률+ROC), 피오트로스키 F-Score.
- **배당주**: 배당성장(배당귀족) + 고배당·저변동성, 분할 정기 적립.
- 전부 Phase 5에서 2TB 예산 내 Parquet로 백테스트 후 규칙 확정 → Phase 6 실행.

## 검증 (End-to-End)
1. `docker compose up` → redis/db/api/collector.
2. collector 로그에서 9거래소+환율+공지 수신, Redis/DB 조회.
3. `GET /premium?base=upbit` → 김프 매트릭스.
4. web → 프리미엄/펀비 대시보드 + 봇 컨트롤 패널 실시간.
5. 텔레그램 알림 + `/bot status`·`/bot start`(페이퍼) 명령.
6. 봇 dry-run 체결 로그(현선/loan/매도/주식).
7. KIS 모의 토큰으로 시세·스크리너·배당 + 일일 브리핑/뉴스요약.
